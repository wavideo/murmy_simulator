# Project Overview

## Concept

이 프로젝트는 머더 미스터리 시나리오를 오프라인 진행용 디지털 보드로 옮긴 React 기반 플레이 툴이다. 현재 기본 시나리오는 `열린 교회 닫힌 문`이며, 플레이어 이동, 밀담 공간 분리, 단서 카드 관리, 스토리북/룰북 열람, GM 진행 보조를 하나의 화면 안에서 처리하는 것을 목표로 한다.

## Product Summary

사용자는 예배당 맵 위에서 캐릭터를 움직이며 대화하고, 단서 카드를 조사·공개·양도하고, 각자 스토리북과 공용 룰북을 확인하면서 추리와 설득을 진행한다. GM은 같은 앱 안에서 방별 상황을 보고, 타이머와 알림을 관리하고, 필요하면 시나리오 데이터 자체도 편집할 수 있다.

## Implementation Stack

- React 19
- Vite 8
- Tailwind CSS 4 (`@tailwindcss/vite`)
- `lucide-react` 아이콘
- 브라우저 `localStorage` 기반 세션 상태/게임 데이터 저장
- 단일 프론트엔드 앱 구조: 별도 서버나 백엔드 없이 클라이언트에서 플레이와 편집 기능을 함께 처리

## Main Features

- 4명의 플레이어 캐릭터와 선택적 GM 캐릭터를 포함한 세션 플레이 상태 관리
- 로비와 밀담 방을 오가는 맵 이동, 방 입장/이탈 피드, 클릭 이동 및 키보드 이동
- 스토리북, 공용 룰북, GM 룰북 열람
- 단서 카드 조사, 뒤집기, 손패 보관, 필드에 재배치, 다른 플레이어에게 양도
- 채팅 입력, 이름 자동완성, 말풍선 및 시스템/GM 피드 표시
- 조사 페이즈용 타이머와 의사결정 세션(지목/투표) 진행 결과 표시
- 커스텀 시나리오 편집 모드: 캐릭터, 룰북 텍스트, 카드, 카테고리, 맵 배치, GM 사용 여부 편집
- 게임 데이터 저장/불러오기/내보내기 및 localStorage 기반 상태 복원

## Structure

- `src/App.jsx`: 사실상 전체 앱의 중심. 기본 시나리오 데이터, 상태 모델, 플레이 UI, 에디터 UI, 이동/카드/투표 로직이 모두 여기에 모여 있다.
- `src/main.jsx`: React 진입점.
- `src/index.css`: 전역 스타일, 커스텀 커서, 채팅/공지 효과, 기본 레이아웃 스타일.
- `assets/`: 배경 이미지, 캐릭터 템플릿 이미지, 오디오, 스프라이트 시트 자산.
- `dist/`: 빌드 산출물.

## Key Flows

- 플레이 시작:
  localStorage에 저장된 상태를 우선 불러오고, 없으면 기본 시나리오 데이터에서 초기 세션 상태를 생성한다.
- 세션 진행:
  활성 플레이어 기준으로 이동, 채팅, 카드 조사, 문서 열람, 방 이동 로그가 갱신된다.
- 카드 상호작용:
  로비에서 필드 카드를 조사해 손으로 가져오고, 손패 카드는 앞/뒷면 전환, 필드 배치, 양도가 가능하다.
- GM 진행:
  GM은 전체 방을 관찰하고, 상단 공지와 타이머를 운영하며, 필요 시 투표/지목 세션 결과를 확인한다.
- 시나리오 편집:
  편집 모드에서 캐릭터/카드/룰북/맵 데이터를 수정하고 저장하거나 JSON으로 입출력한 뒤 그 데이터로 플레이를 재시작할 수 있다.

## Current State

현재 구현은 단일 Vite + React 앱으로 동작하며, 기본 시나리오와 커스텀 시나리오 편집 기능이 함께 들어 있다. 별도 백엔드 없이 브라우저 상태와 localStorage에 의존하며, 핵심 도메인 모델과 UI 로직이 `src/App.jsx` 한 파일에 집중되어 있어 기능 추가나 유지보수 시 결합도가 높은 편이다.

플레이 경험은 이미 꽤 넓게 구현되어 있지만, README와 별도 설계 문서가 거의 비어 있고 테스트 코드도 없다. 따라서 현재 상태를 이해하거나 시나리오를 확장할 때는 코드 자체, 특히 `App.jsx`의 상태 구조와 기본 데이터 정의를 직접 읽어야 한다.
